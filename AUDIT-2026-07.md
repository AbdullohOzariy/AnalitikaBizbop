## Tizim xaritasi (qisqa)

- **Stack:** Next.js 16 (App Router, Server Actions) · React 19 · TS (strict) · Prisma 7 + PostgreSQL (Neon prod / localhost dev, `@prisma/adapter-pg` + `pg` pool) · NextAuth v5 beta (JWT, Credentials + bcryptjs) · Tailwind 4 · telegraf · node-cron. Deploy: Railway. ~44.5k satr TS/TSX, 47 Prisma model, ~40 route/sahifa.
- **Auth oqimi:** `src/proxy.ts` (middleware, supplier.* rewrite) → `src/auth.config.ts` (edge-safe `authorized` callback, `PUBLIC_PREFIXES`) → `src/auth.ts` (Credentials, rol DB'dan 60s kesh bilan). Rol modeli: `src/lib/roles.ts` + `scope.ts` (SYSTEM_ADMIN to'liq, ADMIN "read-only", izolatsiyalangan MERCHANDISER/OPERATOR).
- **Ma'lumot oqimi:** Excel import (`admin/upload`) → `ProductSales`/`CategorySales` → `src/lib/analytics*.ts` (raw SQL agregatlar, `unstable_cache`) → dashboard sahifalar. Miniapp (`bot/miniapp` Vite → `public/miniapp`) initData HMAC bilan `api/*`. Cron hisobotlar `src/instrumentation.ts` (Telegram).

## Umumiy baho: **6.5 / 10**

Kuchli tomonlar: TS strict + toza tsc/eslint/build, auth qatlami puxta (file-upload magic-byte + initData HMAC + rate-limit, rol DB'dan qayta o'qiladi, JWT'ga ishonilmaydi), pg-pool crash himoyasi, kesh-invalidatsiya infratuzilmasi mavjud.
